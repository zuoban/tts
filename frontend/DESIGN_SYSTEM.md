1.  **Semantic Tokens**: We do not use hardcoded colors (e.g., `bg-blue-500`). We use semantic tokens (e.g., `bg-primary`) that map to CSS variables. This enables:
    *   Dark mode support out-of-the-box.
    *   Theming without code changes.
    *   Consistent accessibility.

2.  **Component First**: Styles are encapsulated in components using `class-variance-authority` (cva).
    *   âŒ Avoid: `<button className="bg-blue-500 rounded p-2">`

3.  **Accessibility**: All components must support keyboard navigation (`focus-visible`) and screen readers (`aria-*`).
